Reactive programming w RxJS & Observables

Asynchronous programming paradigm -> data streams & propagation of change/ easier to compose asynchronous or callback-based code (e.g. via next/error/complete). 'The lodash of events'

Creating Observables

  • of(...items) —Returns an Observable instance that synchronously delivers the values provided as arguments.
  • from(iterable) —Converts its argument to an Observable instance. This method is commonly used to convert an array to an observable.
  • Observable constructor e.g. re: custom events

An Observable instance begins publishing values only when someone subscribes to it i.e. by calling the subscribe() method of the instance & passing an observer object to receive the notifications.

  • Converting existing code for async operations into observables

Specifically in Angular:

  • custom events to send observable output data from child component to parent - via EventEmitter
  • AJAX requests/responses w HTTP module
  • Router - user navigation re: ActivatedRoute
  • Reactive Forms: listen for/respond to user input e.g. FormControl properties valueChanges and statusChanges
