Reactive Forms

What & Why?

  • Suited for complex/dynamic forms -> reactive updates -> better UX
  • Uses observables/functional-style ReactiveFormsModule (import from '@angular/forms' & add to app.module.ts imports array) ReactiveFormsModule
  • Logic resides in component, rather than in template -> easier to see (not obscured by CSS classes, HTML attributes etc.)
  • Synchronous, so no need to check what exists before validating
  • Easier to test...

With FormGroup & FormControl

  • Import FormGroup, FormControl from '@angular/forms' into your specific component.ts
  • [formGroup] = directive that connects form to component
  • formGroup = directive that groups nested values inside overall form
  • formControlName = directives that bind inputs to form controls defined inside FormGroup & communicate changes
  • (ngSubmit) = event listener w callback method
  • Button of type submit = triggers submission

Template-Driven Forms

  • Suited for simple forms
  • Arguably a simpler syntax (similar to AngularJS)
  • Data model handled by #formName="ngForm",(ngSubmit)="onSubmit()", and [(ngModel)] & 2-way data-binding - for reading and writing input-control values - ok if you know the shape of your form in advance
  • Asynchronous
  • Uses the FormsModule (import into app.module.ts)
  • Harder to test
